# Release Notes · v0.1.2

发布日期 / Release Date: 2026-02-26

## 中文

### 新增

- 管理后台新增“按用户 + 日期查看情绪旅程图”能力：
  - 用户列表接口：`GET /api/admin/users`
  - 指定用户当日情绪记录接口：`GET /api/admin/users/:userId/mood-entries`
- 后台页面新增“用户日情绪旅程图”模块：
  - 用户选择
  - 日期筛选
  - 曲线查看
  - 当日记录明细（时间、分值、标签、触发词）
- 新增 11 档情绪分值语义（`-5` 到 `+5`）的精细映射与描述。

### 改进

- 评分展示从粗粒度升级为精准标签：
  - 打点表单显示“分值 + 描述”
  - 记录列表显示“分值 + 标签”
  - 曲线图 tooltip 显示“分值 + 标签”
- 词典与风险配置能力增强：
  - 默认词典显著扩充（工作/学业/社交/睡眠/财务/健康/数字过载等）
  - 默认风险词扩充（中英文）
  - 后台保存时自动归一化（去重、去空格、排序、空配置兜底）
- 规则引擎识别增强：
  - 关键词和风险词大小写不敏感
  - scoreBias 累计影响加上限，降低过度偏移

### 质量保障

- 新增并通过以下测试：
  - `packages/ui/__tests__/score-label.test.ts`
  - `apps/web/tests/integration/admin-users-routes.test.ts`
  - `apps/web/tests/integration/data-flow.test.ts`（新增 admin 用户汇总场景）
- 全量 `pnpm typecheck` 与 `pnpm test` 通过。

### 兼容性

- 现有用户侧 API 与核心流程保持兼容。
- 现有后台 metrics/lexicon/risk-terms 接口保持不变。

## English

### Added

- Admin user/day journey inspection support:
  - `GET /api/admin/users`
  - `GET /api/admin/users/:userId/mood-entries`
- New admin dashboard section for:
  - user selection
  - date filtering
  - daily mood curve visualization
  - detailed entries list
- Fine-grained 11-level score semantics (`-5` to `+5`) with labels and descriptions.

### Improved

- UI score semantics upgraded across:
  - entry form
  - entry list
  - chart tooltip
- Lexicon and risk configuration quality improved:
  - richer default lexicon coverage
  - expanded risk terms (CN + EN)
  - normalization on save (trim/dedup/sort/fallback)
- Rule-engine improvements:
  - case-insensitive matching
  - bounded cumulative `scoreBias`

### Quality

- Added tests for score mapping and admin user routes.
- Passed full `pnpm typecheck` and `pnpm test`.

### Compatibility

- No breaking changes to existing user APIs.
- Existing admin metrics/lexicon/risk-terms routes remain compatible.
